# Style Guide

## Typography

Use a clear sans-serif interface font with readable sizes and strong hierarchy between page titles, card titles, labels, and generated content.

## Spacing

Use consistent spacing between controls, cards, and sidebar groups. Avoid dense layouts that make long generated artifacts hard to review.

## Cards

Use existing card patterns for artifacts and status. Cards should have clear titles, readable content, and consistent padding.

## Buttons

Use clear action labels, visible hover states, and disabled/loading behavior when introduced through an approved UX change.

## Colors

Preserve the current dark studio aesthetic and use contrast that keeps text, controls, and status understandable.

## Icons

Use icons sparingly to reinforce meaning. Never rely on an icon alone for an important action or state.

## Responsiveness

Future UI changes should work on common desktop and smaller screen sizes without hiding critical creator controls.

## Accessibility

Use semantic HTML, labels, keyboard-accessible controls, readable contrast, and text equivalents for status icons.

## Consistency

Reuse existing controls, cards, spacing, and naming patterns before introducing visual variants.

## Future Dark Mode

The current UI is dark-first. If themes are added, define shared design tokens and preserve accessible contrast in every theme.
