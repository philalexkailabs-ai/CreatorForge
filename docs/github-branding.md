# GitHub Branding Guide

## Repository Banner

Recommended `assets/branding/banner.png` dimensions: 1600 × 600 pixels. Keep important content centered with generous safe margins.

## Social Preview

Recommended `assets/branding/social-preview.png` dimensions: 1280 × 640 pixels. Include the logo, product name, tagline, and one clear visual cue for local AI content creation.

## Logo Recommendations

- Provide `logo-light.png` for light backgrounds.
- Provide `logo-dark.png` for dark backgrounds.
- Keep a simple recognizable mark for square avatars and favicons.
- Prefer clean geometric shapes over detailed illustrations at small sizes.

## Color Palette

Use the existing Studio direction as the starting point:

- Midnight: `#0F172A`
- Panel: `#1E293B`
- Sidebar: `#111827`
- Accent blue: `#2563EB`
- Light text: `#FFFFFF`
- Muted border: `#2D3748`

## Typography

Use a modern, readable sans-serif family. The current UI uses Segoe UI with Arial fallback. Future repository visuals may use Inter or a similarly legible open font after approval.

## Future Branding Roadmap

- Approve logo system and icon mark.
- Create banner and social preview.
- Capture consistent Studio screenshots.
- Add short demo GIF.
- Define export-safe templates for release announcements and documentation.
